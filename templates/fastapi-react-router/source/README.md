# [PROJECT_NAME]

FastAPI backend + React Router SSR frontend scaffold generated by the SDD template system.

This template is the second proving stack for the refactor. It keeps the reusable workflow and
backend structure from the first reference template while swapping the frontend to React Router
framework mode with route-level metadata for SSR and SEO.

## Quick start

```bash
uv run sdd init --template fastapi-react-router --project-name user-dashboard ./user-dashboard
cd user-dashboard
./scripts/init-project.sh user-dashboard example.com
docker compose up --build
```

See [docs/STACK.md](docs/STACK.md) for setup, commands, and gate expectations.

Optional, when you want compressed agent responses for long debug loops:

```bash
./scripts/install-caveman.sh
```

Then activate only when needed (`/caveman`, Codex: `$caveman`).
